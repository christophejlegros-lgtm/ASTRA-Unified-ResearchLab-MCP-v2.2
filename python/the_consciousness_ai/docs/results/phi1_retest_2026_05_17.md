## Substantive scientific reading (honest)

The mechanical verdict is RE-RUN because phi_std (1.84e-03) and sync_R_std
(0.0193) are below the pre-registered floors (0.01 and 0.02). But the
honest scientific reading is: **the architecture's binding mechanism
(AKOrN phase synchrony) does NOT correlate with the 5-node IIT phi
measurement, even after Phase A+C+D fixes.** Three pieces of evidence:

1. Full-run r = -0.038, statistically significant (p = 7e-14) only because
   of the large sample size (n = 39000). The effect size is essentially zero.

2. Rolling 5000-step windows oscillate between -0.097 and +0.025 with no
   trend. There is no "phase transition" pattern like the 2026-05-16 RIIU
   run showed (it had a transient +0.267 peak that collapsed). This run
   has no peak at all.

3. The architecture is genuinely producing more dynamics than before
   (phi mean 1.82e-03 is 28x larger than the prior 6.5e-05 fixed point,
   sync_R std 0.019 is ~50% larger than the prior 0.013). The dynamics
   exist; they just don't align.

## What this tells us about the architecture

Phase A made phi-on-broadcast structurally downstream of sync_R via
attention-weighted fusion. The fusion machinery works (the synthetic
oscillator gate test in `tests/test_attention_broadcast.py` passes
with |r| > 0.3 on a controlled fixture). But the chain from broadcast
to the IIT phi value is:

  broadcast (256-D) -> ConsciousnessGate forward (5 sigmoid networks)
    -> 5 sigmoid-bounded values
    -> adaptive binarization (now with 1e-5 floor)
    -> 5-bit state -> empirical TPM -> pyphi MIP -> phi scalar

Each transform loses information. The broadcast's covariance structure
with sync_R is real (per Phase A's synthetic test) but does not survive
this chain. By the time we measure phi, the sync_R-correlated signal in
the broadcast has been smeared into the gate's own dynamics and then
compressed into 5 bits.

This is consistent with the 2026-05-16 RIIU finding: when phi was
measured directly on the broadcast (via RIIU, no gate intermediary,
continuous SVD), there WAS a transient r=+0.267. The 5-node gate-state
pyphi pathway is the lossier path.

## Decision options (for the user)

The plan Phase F2 decision table maps RE-RUN to "tune attention_temperature,
audio modulation, mock_semantic salience; do NOT revise threshold." But
the rolling windows show the underlying r oscillates around 0 with no
positive trend — tuning is likely to inflate variance without changing
the correlation. Four honest options:

**Option 1: Tune-and-RE-RUN per the strict pre-registered protocol.**
Try `--attention-temperature 0.25` (sharper softmax -> more bid contrast),
audio amplitude scaling, or higher mock_semantic embedding diversity.
Re-run F2. Cost: ~2 hours wall + ~30 min iteration. Risk: variance goes
above floor but r stays near 0; verdict becomes FAIL.

**Option 2: Escalate to Phase B (content-level binding).** AKOrN-modulated
cross-attention adds an architectural pathway from sync_R into the content
tensors themselves (failure modes 2 and 3 from the diagnosis). The
existing chain would then have sync_R signal in TWO places: post-fusion
broadcast AND inside the content tensors that feed the gate. Cost: ~12h
dev + ~6h F2 re-run. Risk: still no Phi-1 r > 0.4, because the gate-state
TPM is the bottleneck regardless of where sync_R enters.

**Option 3: Switch the phi measurement substrate.** Replace pyphi
(5-node gate TPM) with RIIU on broadcast (continuous SVD), which the
2026-05-16 result showed has shorter measurement path. The Phase A+C+D
fixes were not present then, so the prior r=+0.075 / transient +0.267
may have been an underestimate. Cost: just re-run F2 with `--enable-riiu`
added, ~2h. The Phi-1 prediction was always specifically about pyphi,
so this is a sub-question: does RIIU under the new architecture clear
the threshold, even though pyphi doesn't?

**Option 4: Accept and publish.** Document this as: "the gate-state pyphi
pathway, even with structurally downstream broadcast (Phase A), gate-collapse
fixes (Phase C), and multi-modal binding (Phase D), does not exhibit
Phi-1 correlation. The architecture's binding signal does not propagate
to the IIT measurement substrate." This is a real scientific finding.
Proceed to Phase 5 of `docs/roadmap.md` with this on record.

Recommendation: Option 3 first (cheapest, directly tests whether the
new architecture + RIIU produces what pyphi can't), then Option 4 with
the combined result. Option 1 risks gaming variance; Option 2 is large
work without clear evidence the gate-state pathway can ever carry the
signal.
