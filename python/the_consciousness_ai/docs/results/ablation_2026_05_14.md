# Ablation Campaign 2026-05-14: Peaceful Castle (6 runs)

This document is the permanent record of the 6-run "peaceful castle" ablation
campaign that ran 2026-05-13 22:28 → 2026-05-14 06:26 wall time. Each run
ablates one architectural component to isolate its contribution to phi
dynamics and reward.

All numbers are loaded from disk in the same session that wrote this doc, per
the CLAUDE.md NEVER-LIE protocol. Source: `runs/ablation/<run>/metrics.csv`,
`runs/ablation/<run>/episodes.csv`, and
`scripts/analysis/ablation_report.py`.

## Setup

- **Environment:** `dark_room` (SimpleVisualEnv, 224×224 frames)

- **Max steps per episode:** 200 (40,000 steps per run)
- **Common flags:** `--phi-sample-every 5 --log-ei-every 50`

  contention)
- **Wall clock:** ~1h 33m per run on average; ~9 hours total for the campaign
- **Master log:** `runs/ablation/_master.log`
- **Per-run completion timestamps** (from master log):

## D_no_consfix: expected crash

D crashed at episode 1 with this traceback (excerpt from
`runs/ablation/D_no_consfix.log`):

```
File "models/memory/memory_core.py", line 367, in _calculate_emotional_strength
    valence = abs(exp["emotion"].get("valence", 0.0))
KeyError: 'emotion'
```

This is the **exact regression that commit `deef672` fixed** when it changed
`MemoryConsolidationManager._merge_similar()` to preserve all fields
(emotion, state, action) on merge. The `--ablate-consolidation-fix` flag
reverts that change. The crash confirms the fix is load-bearing: without
it, training cannot proceed past the first memory consolidation cycle.

D is therefore reported as **MISSING DATA**, not as a failed run. The
finding from D is structural (the fix is required), not metric.

### EI-1 (workspace EI > gate EI first appears between episodes 500-2000)

All 5 runs: **FAIL — first emergence at episode 49.**

This is an artifact of the logging cadence (`--log-ei-every 50` means the
first EI measurement happens at episode 49) combined with the fact that
EI(workspace) > EI(gates) on the very first measurement. The prediction
that emergence appears **between 500-2000** was set before measurements
were taken; the artifact-driven early emergence is a real FAIL regardless
of cause.

### IM-1 (phi spike > 1.5 SD at insight moments)

All 5 runs: **FAIL**. Insight phi means range from 7e-04 to 1.1e-03; the
threshold (1.5 SD above each run's own phi mean) is just slightly above
those values, and insight moments are not concentrated in the high-phi tail.

### IM-3 (sync_R > 0.7 at insight moments)

All 5 runs: **FAIL — 0.0% of insights have R > 0.7.** sync_R during these
runs stays in the [0.21, 0.24] range; the 0.7 threshold is far above any
observed value.

times per run, too few for ratio stabilization or correlation tests) or

## Counter-intuitive findings (single-seed observations)

The following observations contradict design assumptions in the codebase.
**Each is from a single seed.** RNG noise is a plausible alternative
explanation, so none should drive code changes without ≥3 seeds confirming
the direction.

### 1. Removing the per-neuron diversity loss increases phi_std

| | phi_mean | phi_std |
|---|---|---|
| A_current (per-neuron loss ON) | 7.72e-04 | 3.45e-04 |
| E_no_div (per-neuron loss OFF) | 1.07e-03 | 1.17e-03 |

E_no_div has 38.7% higher phi_mean and 240% higher phi_std than the head.
This contradicts the 5-episode smoke test from 2026-05-13 that showed
per-neuron diversity loss was helpful. The 200-episode result is the
trustworthy comparison.

### 2. The RND zero-on-reward guard appears to hurt reward

| | last100 reward | positive eps |
|---|---|---|
| A_current (guard ON) | +8.383 | 62 |
| G_no_rnd_zero (guard OFF) | +18.897 | 71 |

G has 2.25× higher last-100 reward and 14.5% more positive-reward episodes
than the head. The guard was added in commit `eeaec28` with the rationale
"prevent agent from leaving rewarding states in dark_room". This single-seed
result suggests the opposite: letting RND fire on rewarded states improves
overall behavior. Reward jumps in F_no_fb (+14.1) and G_no_rnd_zero (+18.9)
both come from runs where the head's behavioral biases are removed.

### 3. The gate_feedback projection gives the worst Phi-1 r (when present)

| | Phi-1 r |
|---|---|
| A_current (gate_feedback ON) | +0.0598 |
| F_no_fb (gate_feedback OFF) | +0.0887 |

F_no_fb has the highest Phi-1 r of all 5 runs. The cross-step
`gate_feedback` projection in `ConsciousnessGate.forward` was added to
match the GATE_CM connectivity, but may be subtly anti-correlating phi with
sync_r at the data level.

### 4. Memory replay has negligible effect

| | phi_mean | last100 reward | Phi-1 r |
|---|---|---|---|
| A_current (replay ON) | 7.72e-04 | +8.383 | +0.0598 |
| C_no_replay (replay OFF) | 7.25e-04 | +9.969 | +0.0225 |

The C-vs-A delta is the smallest of any flag pair. Memory replay does not
appear to be the load-bearing component for either phi dynamics or reward.

## Verdict against decision protocol

[preregistered_predictions.md § 5](../preregistered_predictions.md) lists
five protocol outcomes. The campaign's result maps to:

> **4. Both fail:** The architecture does not produce measurable emergence
> or integration. Fundamental redesign needed (different binding mechanism,
> different gate structure, or abandon the strong emergence claim and
> reframe as weak emergence).

Both EI-1 and Phi-1 fail in every variant. The campaign confirms that the
phi-sync_R relationship is not a tunable hyperparameter or single
component's contribution; it is intrinsic to the current
ConsciousnessGate + AKOrN + GATE_CM stack.

## Hypotheses for future work

Each item below is a hypothesis flagged by the single-seed counter-intuitive
findings above. **None should be merged without ≥3 seeds confirming the
direction.**

1. **Drop per-neuron diversity loss.** E_no_div had higher phi_mean and
   phi_std. Test by removing the `-log(|g-0.5|)` block at
   [scripts/training/train_rlhf.py:578-587](../../scripts/training/train_rlhf.py#L578-L587)
   and running ≥3 seeds at 200 episodes.
2. **Drop the gate_feedback projection.** F_no_fb had the best (least bad)
   Phi-1 r. Test by setting `ablate_gate_feedback=True` as default in
   [models/core/consciousness_gating.py](../../models/core/consciousness_gating.py).
3. **Drop the RND zero-on-reward guard.** G_no_rnd_zero had the highest
   reward. Test by removing the `if env_reward > 0` condition around the
   curiosity_score zeroing in
   [scripts/training/train_rlhf.py](../../scripts/training/train_rlhf.py).
4. **Try RIIU integration** as a fundamentally different phi formulation.
   See arxiv 2506.13825 (Apache-2.0 on the code; the paper text is
   CC BY-NC-SA 4.0). The current pyphi-based pipeline produces phi values
   in the 1e-4 to 1e-3 range; RIIU's differentiable Auto-Phi with
   sliding-window covariance produces phi values that may be more
   responsive to short-term binding events.
   **TESTED AND FAILED 2026-05-16.** Single-seed 200-episode comparison
   run (`runs/riiu_compare_seed42`) gave full-run r(phi_riiu, sync_R) =
   +0.0754, worse than pyphi's best of +0.089. A transient r=+0.267 peak
   appeared during steps 11000-16000 then collapsed. RIIU phi stayed near
   2.75e-04 (below the 1e-3 signal-alive floor). Full verdict in
   [riiu_compare_2026_05_16.md](riiu_compare_2026_05_16.md). RIIU code
   kept merged behind `--enable-riiu` (default off) as a diagnostic.

## Reproducibility

Every number in this document can be reproduced by:

```bash
# Verify run completion
cat runs/ablation/_master.log

# Re-run the ablation report
python scripts/analysis/ablation_report.py

# Re-run per-prediction verdicts for any run
python -m scripts.analysis.analyze_experiment --run-dir runs/ablation/A_current
```

The raw `runs/ablation/` data is gitignored by repo policy (per
`.gitignore`). To reproduce from scratch:

```bash
PYPHI_WELCOME_OFF=yes python -m scripts.training.train_rlhf \
  --episodes 200 --max-steps 200 --env dark_room \
  --phi-sample-every 5 --log-ei-every 50 \
  --log-dir runs/ablation/A_current
```

substituting the appropriate `--ablate-*` flag per row in the "Runs" table.
